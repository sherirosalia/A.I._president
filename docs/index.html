A tale of tweets and two Markov chains

Trump's tweets put through "algorithmic paces"

Stoatistic algorithms and natural language processing applied to tweet generation.

Curious as to whether or not machine learning would be able to mimic natural language, it seeemed like twitter and one of it's iconic members could be a great source of source data. Approximately 50 pages of tweets along with the application of two different programming scripts in an attempt to analyze and mimic President Trumps tweets produced fairly succssful and sometimes hilarious results.

The first step was to get the data

Why copy and paste when the computer can do it for us? The video below shows how a browser can be automated to scroll through Twitter. The entire process of scrolling through recent history took about 20 minutes and in the background the script was making choices as to what information to keep and which posts to skip. For example, the decision was made to skip over re-tweets. For those who are unfamiliar with Twitter, "re-tweets" are usually recycled posts from other users and can therefore dilute what could be attributed to the President.

Putting the data through an analysis library

A Markov Chain breaks analyzes word frequencies and patterns to assess likely word orders. The first step was to run the data from twitter through an "off the shelf" natural language processing package called "Markovify" which provided a convenient way to assess the likelihood of successful imitation. Check out the screenshot below and/or the table of full results.

Custom algorithm

The next step was to create a custom Markov Chain script to compare the results with the and in the end, both procesees generated tweets with a Trump-like flavor with a slap-stick style humor. Check out the final results of the custom script here: the table of full results.

The next step

If the project was to be developed further, more data is the first step I would take as the results were almost "too good", especially for the markovify package results. Additonally, the custom script would benefit from some conditional clauses to normalize the results. For example, the custom script results had quite a few stray characters that were not produced using the Python natural language processing library.
